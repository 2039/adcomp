3 Simulation {#Simulation}
4 ==========
5 
6 When building models in TMB it is generally recommended to test the
7 implementation on simulated data. Obviously, data can be simulated
8 from R and passed to the C++ template. In practice this amounts to
9 implementing the model twice and is thus a strong way to validate the
10 implementation of the model. However, with increased model complexity
11 it becomes inconvenient to maintain two separate
12 implementations. Therefore, TMB allows the the user to write the
13 simulation code as an integrated part of the C++ model template.
14 
15 
16 ## Overview of simulation methods in TMB
17 
18 ### Standard generators
19 
20 The TMB simulation routines use the same naming convention as the R
21 simulators. For instance `rnorm()` is used to simulate from a normal
22 distribution. However, the argument convention is slightly
23 different:
24 
25 1. `rnorm(n, mu, sd)` draws `n` simulations from a normal
26  distribution. Unlike R this works for **scalar parameters only**.
27 2. `rnorm(mu, sd)` is a TMB specific variant that works for
28  **mixed scalar and vector input**. Output length follows the length of the
29  longest input (no re-cycling) hence is consistent with `dnorm(mu, sd)`.
30 
31 Currently the following simulators are implemented:
32 
33 > `rnorm()`, `rpois()`, `runif()`, `rbinom()`, `rgamma()`, `rexp()`, `rbeta()`, `rf()`, `rlogis()`, `rt()`, `rweibull()`, `rcompois()`, `rnbinom()`, `rnbinom2()`
34 
35 ### Generators for density objects
36 
37 Objects from the density namespace have their own `simulate()`
38 method. Taking the multivariate normal distribution as example we have
39 the following ways to draw a simulation:
40 
41 1. `MVNORM(Sigma).simulate()` returns a vector with a simulation from
42  the multivariate normal distribution. The void argument version is
43  only available when there is no ambiguity in the dimension of the
44  output. In the `MVNORM` case the dimension of the output is known
45  from the dimension of `Sigma`. In other cases e.g. `AR1(phi)` the
46  dimension of the output is not known hence the void argument
47  version is not available.
48 2. `MVNORM(Sigma).simulate(x)` pass `x` by reference and writes the
49  simulation directly to `x` without returning anything. This version
50  is available for *all* the classes because the dimension of the
51  simulation can always be deduced from `x`.
52 
53 ## Simulation blocks
54 
55 Simulation functions can be called from anywhere in the C++
56 program. However, usually one should put the simulation code inside
57 specialized *simulation blocks* that allows the code to only be
58 executed when requested from R.
59 
60 ### A linear regression example
61 
62 A complete example extending the example \ref linreg.cpp
63 with simulation code is:
64 
65 
66 
67 
68 ```{cpp}
69 
70 #include <TMB.hpp>
71 template<class Type>
72 Type objective_function<Type>::operator() ()
73 {
74  DATA_VECTOR(y);
75  DATA_VECTOR(x);
76  PARAMETER(a);
77  PARAMETER(b);
78  PARAMETER(sd);
79  vector<Type> mu = a + b * x;
80  Type nll = -sum(dnorm(y, mu, sd, true));
81  SIMULATE {
82  y = rnorm(mu, sd); // Simulate response
83  REPORT(y); // Report the simulation
84  }
85  return nll;
86 }
87 
88 ```
89 
90 
91 > The `SIMULATE` block marks the simulation and is not executed by default.
92 
93 
94 
95 We compile the C++-file and the model object is constructed as usual:
96 
97 
98 ```r
99 obj <- MakeADFun(data, parameters, DLL="linreg")
100 ```
101 
102 Now a simulation can be generated with
103 
104 
105 ```r
106 obj$simulate()
107 ```
108

113 ```
114

135 ### Advanced examples
136 
137 The examples \ref sam.cpp and \ref ar1_4D.cpp
138 includes more advanced simulation code. The latter demonstrates how to
139 simulate from the density objects:
140 
141 
142 ```{cpp}
143 // Separable covariance on 4D lattice with AR1 structure in each direction.
144 #include <TMB.hpp>
145 
146 /* Parameter transform */
147 template <class Type>
148 Type f(Type x){return Type(2)/(Type(1) + exp(-Type(2) * x)) - Type(1);}
149 
150 template<class Type>
151 Type objective_function<Type>::operator() ()
152 {
153  DATA_VECTOR(N)
154  PARAMETER_ARRAY(eta);
155  PARAMETER(transf_phi); /* fastest running dim */
156  Type phi=f(transf_phi);
157  ADREPORT(phi);
158 
159  using namespace density;
160  Type res=0;
161 
162  res+=AR1(phi,AR1(phi,AR1(phi,AR1(phi))))(eta);
163 
164  // logdpois = N log lam - lam
165  for(int i=0;i<N.size();i++)res-=N[i]*eta[i]-exp(eta[i]);
166 
167  SIMULATE {
168  AR1(phi,AR1(phi,AR1(phi,AR1(phi)))).simulate(eta);
169  vector<Type> lam = exp(eta);
170  N = rpois(lam);
171  REPORT(eta);
172  REPORT(N);
173  }
174 
175  return res;
176 
177 }
178 ```
179 
180 
181 In this example the 4D-array `eta` is passed to the simulator by
182 reference. Thereby the simulator knows the dimension of `eta` and can
183 fill `eta` with a simulation.
